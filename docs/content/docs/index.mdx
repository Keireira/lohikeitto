Lohikeitto (Soup) is a service catalog API. It powers subscription search in Sharkie — users type a service name and get back logos, domains, and metadata from multiple sources.

Full endpoint documentation is available in the [API Reference](/api-reference).

## How search works

When a client sends `GET /search?q=spotify`, the API searches up to three sources **in parallel**:

```
           ┌─── Local DB (PostgreSQL, pg_trgm) ──── ~50ms
           │
 request ──┼─── Brandfetch API ───────────────────── ~500ms
           │
           └─── logo.dev API ─────────────────────── ~800ms
```

The `sources` parameter controls which sources are queried — as a comma-separated list:

```
/search?q=spotify&sources=local,brandfetch
```

Aliases: `external` (brandfetch + logodev), `all` (default — all three).

Results are **deduplicated by domain** with priority:

```
local  >  brandfetch  >  logo.dev
```

If `spotify.com` appears in all three — only the local version is kept.

Local (curated) services may have multiple domains — all of them are searchable and each one blocks external duplicates. External results always have a single domain and are **never merged** with each other to prevent search poisoning.

### Frontend strategy

The client fires two sequential requests to keep the UI responsive:

1. `GET /search?q=...&sources=local` — renders instantly (~50ms)
2. `GET /search?q=...&sources=external` — appends results when ready (~1-2s)

External results use a **deterministic UUID v5** derived from the domain, so React keys stay stable across requests.

## Quick Start

```bash
cp lohikeitto.env.example lohikeitto.local.env
# fill in API keys (Brandfetch, logo.dev, Cloudflare R2)

make dev
```

Migrations run automatically on startup. The database is created if it doesn't exist.
