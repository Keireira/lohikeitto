## Public endpoints

| Endpoint | Description |
| --- | --- |
| [`GET /search?q=`](/docs/api/search) | Fuzzy search by name with Brandfetch fallback |
| [`GET /services/:id`](/docs/api/services) | Service details by UUID |
| [`GET /init?country=`](/docs/api/init) | Preload services available in a country |
| [`GET /health`](/docs/api/health) | Health check (DB liveness) |

## Admin endpoints

All admin endpoints require `Authorization: Bearer <ADMIN_TOKEN>` header. The admin panel (`make admin`) provides a UI for these endpoints. See [Admin API](/docs/api/admin) for details.

| Endpoint | Description |
| --- | --- |
| `GET /admin/services?filter=` | List services |
| `GET /admin/services/:id` | Full service details |
| `PUT /admin/services/:id` | Update service |
| `DELETE /admin/services/:id` | Delete service |
| `PUT /admin/services/:id/approve` | Approve service |
| `POST /admin/services/:id/sync-logos` | Sync logos for service |
| `POST /admin/services/:id/save-logo` | Save a single logo |
| `GET /admin/categories` | List categories |
| `GET /admin/export-sql?filter=` | Export services as SQL |

## Response format

All responses use a unified JSON envelope:

```json
// Success
{ "status": "success", "data": T }

// Error
{ "status": "error", "code": N, "message": "..." }
```
