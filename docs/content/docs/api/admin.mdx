All admin endpoints require `Authorization: Bearer <ADMIN_TOKEN>` header. Returns 401 without a valid token.

The admin panel (`make admin`) provides a UI for these endpoints on port 3001.

## Services

### GET /admin/services

List services with optional filter.

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `filter` | string | No | `all` (default), `verified`, `unverified` |

### GET /admin/services/:id

Full service details for admin panel.

### PUT /admin/services/:id

Update service fields: `name`, `slug`, `domain`, `category_id`, `colors`, `links`, `countries`, `ref_link`.

### DELETE /admin/services/:id

Delete a service.

### PUT /admin/services/:id/approve

Set `verified = true`.

## Logos

### POST /admin/services/:id/sync-logos

Download logos from Brandfetch and logo.dev for a specific service, upload to R2.

### POST /admin/services/:id/save-logo

Save a single logo type (`bf_logo`, `bf_symbol`, `logodev`) to R2.

## Categories

### GET /admin/categories

List all categories.

## Export

### GET /admin/export-sql

Export services as SQL INSERT statement.

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `filter` | string | No | `all`, `verified`, `unverified` |
| `ids` | string | No | Comma-separated UUIDs |

Returns the SQL in the response `data` field.
